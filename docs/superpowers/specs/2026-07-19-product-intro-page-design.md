# 功能介绍主页设计方案

## 目标

为“习惯养成”新增一个面向家庭用户的独立功能介绍页，用清晰的产品价值、真实界面截图和简短使用流程，引导新用户进入现有任务系统。

介绍页使用 `intro.html`，现有 `index.html` 继续作为任务系统入口，不调整现有业务路由、数据模型或交互逻辑。

## 目标用户与核心行动

- 目标用户：希望帮助孩子建立日常习惯、合理管理 iPad 使用时间的家庭。
- 页面目标：让首次访问者快速理解产品解决什么问题、包含哪些功能以及实际界面效果。
- 核心行动：点击“立即开始”进入现有任务系统 `index.html`。

## 视觉方向

采用已确认的“A 方案：产品型双栏首屏”。

- 参考 inkCloudNote 展示站的大留白、细边框、清晰锚点导航和真实截图卡片。
- 延续现有项目的紫色品牌色，但降低大面积高饱和背景的使用，让页面更轻盈、温和，符合家庭产品气质。
- 首屏左侧承载价值主张与行动按钮，右侧承载真实产品截图，用户无需滚动即可同时理解产品与看到界面。
- 使用统一圆角、柔和阴影和低对比度分区背景建立层级，不使用与当前产品风格冲突的装饰语言。

## 页面信息结构

### 1. 顶部导航

- 品牌名称：“习惯养成”。
- 锚点入口：功能介绍、产品截图、使用场景。
- 主要按钮：“立即开始”，链接到 `index.html`。
- 桌面端保持单行；手机端简化锚点入口，保留品牌与主要按钮。

### 2. 首屏

- 标签：家庭习惯与时间管理。
- 主标题：“让好习惯，在家里自然发生”。
- 副标题：说明产品把每日打卡、家庭协作和 iPad 使用管理集中到一个清晰的家庭任务系统中。
- 主按钮：“立即开始”。
- 次按钮：“查看功能”，滚动到核心功能区域。
- 右侧展示任务日历与当天任务的真实产品截图。

### 3. 核心功能

使用三张功能卡片介绍：

1. 每日任务打卡：支持单次、每日和每周任务，日历直观展示完成状态。
2. 家庭成员协作：家庭邀请码、多成员头像与视图切换，让每个人拥有自己的任务日历。
3. iPad 使用管理：设置每日额度、记录使用时段、区分计入和不计入额度的类型，并提示超时。

### 4. 产品截图

展示三类真实界面：

- 日历与每日任务视图。
- 任务完成备注、图片凭证和详情。
- iPad 使用额度、实时计时、使用记录与超时日历。

截图需保存为项目内的优化图片资源，避免依赖外部附件链接。每张图片提供清晰的说明文字和替代文本，并声明宽高或使用固定宽高比，避免加载时页面跳动。

### 5. 使用场景

用四步展示产品使用流程：

1. 创建或加入家庭。
2. 为当前成员制定任务和 iPad 使用额度。
3. 每日完成任务并留下备注或照片。
4. 在日历中回顾完成情况与用屏时间。

### 6. 页尾行动区

- 强调“从今天开始，让成长看得见”。
- 再次提供“立即开始”按钮。
- 简短说明数据支持家庭共享与云端同步。

## 响应式与可访问性

- 桌面端首屏采用双栏布局；窄屏改为文字在上、截图在下。
- 页面不产生水平滚动，截图在容器内自适应缩放。
- 所有按钮和链接的触控区域不小于 44 像素，并提供清晰的键盘焦点样式。
- 正文保持足够对比度，正常文本不小于 16 像素。
- 使用正确的标题层级、导航语义、图片替代文本和跳到主要内容链接。
- 动效只用于内容进入和按钮反馈，时长控制在 150–300 毫秒，并尊重 `prefers-reduced-motion`。

## 技术边界

- 新增 `intro.html` 和介绍页专属样式文件，必要时添加少量只负责导航或展示动效的脚本。
- 介绍页不加载 Supabase，不读取家庭数据，不改动现有任务系统状态。
- `index.html` 仍是产品应用入口，介绍页所有“立即开始”链接均指向它。
- 不引入新的前端框架或构建依赖，继续使用静态 HTML、CSS 和原生 JavaScript。
- 新增结构测试，验证页面标题、主要区域、行动链接、截图替代文本和响应式样式契约。

## 验证标准

- `intro.html` 可以通过本地 HTTP 服务独立打开。
- 桌面和手机尺寸下无溢出、遮挡或不可点击元素。
- 页面至少包含三项功能介绍和三类产品截图。
- 首屏与页尾的“立即开始”都能进入 `index.html`。
- 现有任务系统测试全部通过，介绍页不会触发 Supabase 请求或修改本地数据。
