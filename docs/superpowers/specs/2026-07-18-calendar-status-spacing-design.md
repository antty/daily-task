# 日历选中态与任务状态间距设计

## 目标

解决任务日历日期被选中后，日期徽标与完成状态 `✓` / 未完成状态 `✕` 距离过近的问题，同时保持现有温柔薰衣草视觉体系。

## 设计方案

- 选中日期数字继续使用左上角紫色圆角徽标，作为主要选中反馈。
- 任务状态固定在日历格右下角，与日期徽标形成清晰的对角信息层级。
- 状态图标与格子边缘保持 6–8px 内边距，和日期徽标之间保持足够留白。
- 有任务但尚未形成完成结果的浅绿色短横同样定位在底部区域，并保持低视觉权重。
- 空闲日期状态保持弱化，不额外强调。
- 手机端不增加日历格高度，不引起日历整体跳动或横向溢出。

## 技术范围

- 仅修改日历状态相关 CSS 定位与间距。
- 不修改任务状态判断、日期选择事件、Supabase 数据或任务业务逻辑。
- 继续使用当前 HTML 语义及 `aria-label`，不削弱辅助技术可读性。

## 验证标准

- 选中日期同时存在 `✓` 或 `✕` 时，两者不接触、不重叠。
- 完成、未完成、有任务和空闲四种状态均保持可识别。
- 桌面与小屏布局中日历格尺寸稳定。
- 自动化测试覆盖选中日期和状态图标采用独立定位区域。
