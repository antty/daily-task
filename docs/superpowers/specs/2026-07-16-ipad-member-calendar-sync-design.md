# iPad 成员与日历联动设计

## 目标

修正 iPad 使用管理在切换家庭成员后残留旧任务的问题，并让日历成为使用记录的浏览入口。

## 交互

1. 切换成员时，将 iPad 视图的选中日期重置为当天，并按新成员与该日期重新渲染额度、记录和汇总。
2. iPad 日历中的日期可点击。点击后更新选中态，并在左侧展示该日期的使用记录。
3. 左侧记录区域显示完整日期标题；无父任务时显示该日期没有创建使用额度的空状态。
4. 仅当天允许创建父任务或新增使用记录；历史日期仅供查看。

## 数据流

`state.memberId` 与 `state.ipadSelectedDate` 是 iPad 页面查询上下文。渲染函数以两者过滤 `ipad_daily_limits` 和 `ipad_usage_entries`，而不再固定使用当天日期。成员切换统一重置 `ipadSelectedDate`，日历点击只更新该日期。

## 边界与验证

- 切换成员不会变更既有任务归属或数据。
- 新成员没有当日任务时，不能看见上一位成员的额度或记录。
- 点击历史日期可以看到该日记录，但创建按钮禁用并给出提示。
- 自动化测试覆盖成员切换重置和日历日期驱动的渲染结构；全量测试需保持通过。
