# 页面视觉统一与家庭管理密码设计

日期：2026-07-17  
状态：已确认，待实施

## 1. 目标

在不改变每日任务、日历和 iPad 计时核心业务的前提下，统一首页、管理弹窗和 iPad 使用管理页的视觉细节，并将目前硬编码在前端的家庭管理密码改造成可修改、可跨设备同步的家庭级管理密码。

本次同时为 iPad 每日额度增加 `185 分钟`预设，最终预设为 `60、120、180、185 分钟`，继续保留自定义分钟数。

## 2. 范围

### 2.1 视觉优化范围

- 首页顶部、日历与任务列表。
- 家庭成员选择、身份验证、家人管理和切换家人弹窗。
- 任务列表、任务类型、添加与编辑任务相关弹窗。
- iPad 使用管理页、使用类型、添加记录、完成记录和额度创建区域。
- 桌面端与手机端的响应式布局、触控区域及弹窗层级。

视觉调整只处理布局、层级、间距、字号、颜色、阴影、状态反馈和响应式表现，不改变现有业务字段、任务归属、完成规则和 iPad 超时算法。

### 2.2 密码功能范围

- 家人管理与 iPad 使用类型管理共用同一个家庭管理密码。
- 初始密码保持为 `123456`，兼容现有家庭。
- 密码修改后，同一家庭的所有已加入设备同步生效。
- 任何知道当前密码且已经加入该家庭的人都可以修改密码。
- 修改时必须填写当前密码、新密码和确认密码。

## 3. 视觉设计

### 3.1 统一视觉规则

- 收敛分散的重复样式，建立统一的颜色、圆角、阴影、间距和控件高度变量。
- 页面内容使用一致的最大宽度和左右留白，避免不同页面主体宽度跳变。
- 标题、说明、辅助信息建立清晰的三级文字层级，减少无意义的英文装饰和过大的标题。
- 主要按钮、次要按钮、文字按钮和危险按钮保持明确区分；关闭按钮弱化但保留足够触控面积。
- 输入框、选择框和按钮的可交互高度在手机端不低于约 44px。
- 错误、成功、进行中、完成和超时状态使用稳定的颜色与图标组合，不只依赖颜色表达。
- 弹窗使用一致的蒙层、圆角、阴影、标题区域和操作区；嵌套弹窗打开时保持清晰的层级。

### 3.2 页面重点

- 首页保持头像、昵称、激励文字和入口按钮的单行重心，避免小屏幕操作入口换行。
- 日历与任务列表在桌面端对齐，手机端自然纵向排列。
- 家庭成员选择强调头像和昵称，管理入口保持次要层级。
- 家人管理、任务管理和类型管理减少无效留白，让列表与主要操作更容易扫描。
- iPad 使用管理保持额度摘要、记录列表和日历的视觉关联；超时信息继续使用醒目的红色状态，但避免大面积警示色造成压迫。

## 4. 密码安全设计

### 4.1 存储

新增私有 schema 中的家庭管理密钥表，按家庭保存一条密码哈希。前端不能直接查询该表。

- 不保存明文密码。
- 使用 PostgreSQL `pgcrypto` 的 `crypt()` 和带随机盐的 `gen_salt('bf')` 生成哈希。
- 现有家庭在迁移时创建初始密码 `123456` 对应的独立随机盐哈希。
- 新建家庭通过数据库逻辑自动获得初始密码哈希。

### 4.2 服务端接口

提供两个受限 RPC：

1. `verify_household_management_password`
   - 输入家庭 ID 和待验证密码。
   - 先确认当前 Supabase 用户能够访问该家庭。
   - 只返回布尔验证结果，不返回哈希及其他密钥信息。

2. `change_household_management_password`
   - 输入家庭 ID、当前密码和新密码。
   - 验证调用者属于该家庭且当前密码正确。
   - 校验新密码规则后生成新的随机盐哈希并更新。
   - 返回明确的成功或受控错误结果。

两个函数使用固定且最小化的 `search_path`，撤销 `public` 与 `anon` 的执行权限，只向 `authenticated` 角色授权。家庭数据的实际隔离继续由 Supabase Auth 与 RLS 负责。

### 4.3 前端会话

- 浏览器只把密码提交给 RPC，不将密码或哈希写入 localStorage、sessionStorage、URL、日志或状态持久化数据。
- 验证成功状态仅保存在当前页面的内存中。
- 页面刷新、页面关闭或切换家庭后，管理验证状态立即失效。
- 家人管理和 iPad 使用类型管理统一调用同一验证方法，移除两处 `password !== '123456'` 硬编码。

### 4.4 密码规则

- 新密码为 6–12 位纯数字。
- 新密码与确认密码必须一致。
- 新密码不得继续使用 `123456`。
- 禁止全部数字相同，例如 `111111`。
- 禁止完整递增或递减连续数字，例如 `1234567`、`765432`。
- 错误提示使用中文用户文案，不直接展示数据库内部错误。

## 5. 密码交互

家人管理页新增弱化的“修改管理密码”入口。点击后打开独立修改弹窗：

1. 输入当前密码。
2. 输入新密码。
3. 再次输入新密码。
4. 前端先检查格式与一致性。
5. 调用 Supabase 修改 RPC。
6. 成功后关闭弹窗并展示成功提示；所有设备下次验证时使用新密码。

验证失败时保留弹窗并聚焦到当前密码；网络或 Supabase 错误显示可重试提示。提交期间按钮进入禁用和加载状态，防止重复提交。

## 6. iPad 额度调整

- 默认额度按钮为 `60 分钟`、`120 分钟`、`180 分钟`和`185 分钟`。
- 自定义分钟数继续支持 1–600 分钟。
- 185 分钟与其他预设共用同一创建逻辑，不新增特殊计算分支。
- 额度创建后的剩余时间、超时统计和日历状态继续使用现有领域逻辑。

## 7. 数据与代码边界

- `supabase/schema.sql`：包含全量新项目结构。
- 新增独立密码迁移 SQL：供现有 Supabase 项目执行，不重复破坏既有数据。
- `src/supabase-store.js`：封装密码验证和修改 RPC，不向 UI 暴露 Supabase 查询细节。
- `src/task-store.js`：提供离线或无 Supabase 环境下的兼容接口，默认密码只用于本地回退测试，不持久化明文到业务状态。
- `src/app.js`：负责弹窗状态、表单校验、加载态和提示，不直接实现哈希逻辑。
- HTML/CSS：统一视觉组件与响应式规则，业务渲染逻辑保持解耦。

## 8. 错误处理

- RPC 返回密码错误时展示“当前密码不正确”。
- 新密码不符合规则时在请求前展示具体规则错误。
- 网络失败时展示“暂时无法连接，请稍后重试”，不错误地进入管理页面或关闭修改弹窗。
- 数据库未执行迁移时展示明确的配置提示，避免继续使用前端默认密码形成安全降级。
- UI 写入成功提示只能在服务端请求成功后展示。

## 9. 测试与验收

### 9.1 自动化测试

- 前端不再出现硬编码密码比较。
- 家人管理与 iPad 使用类型管理都调用统一密码验证接口。
- 新密码格式、一致性和弱密码规则测试。
- 修改成功、当前密码错误、网络错误的交互测试。
- SQL 文件包含私有哈希表、访问校验、权限撤销和两个 RPC。
- iPad 额度包含 185 分钟预设，并复用现有额度创建方法。
- 页面资源引用使用同一个新缓存版本号。
- 运行全部 Node 测试、JavaScript 语法检查和 `git diff --check`。

### 9.2 手动验收

- 桌面端检查首页、家庭相关弹窗、任务管理和 iPad 页面。
- 手机宽度检查弹窗不超屏、按钮不拥挤、关闭与返回入口可操作。
- 使用初始密码进入两个受保护入口。
- 修改密码后，旧密码失效，新密码在另一台已加入家庭的设备生效。
- 刷新页面后必须重新验证。
- 创建 185 分钟额度，并验证剩余、完成和超时展示正常。

## 10. 发布约束

- 所有 CSS 和 JavaScript 静态资源统一更新缓存版本号，避免手机继续使用旧样式或旧逻辑。
- SQL 迁移必须先在 Supabase 执行，再发布依赖新 RPC 的前端版本。
- 本次在功能分支开发；测试完成后，合入主分支和上线前再次取得用户同意。
